# تعليمات وكيل التطوير (Developer Agent Instructions)

هذا الملف يحتوي على القواعد والتعليمات الخاصة بوكيل الذكاء الاصطناعي (AI Agent) للعمل على مشروع "غيابي".

## 🎯 مهام الوكيل (Agent Tasks)
- **إدارة مساحة العمل**: التعامل مع المجلد الجذري للمشروع فقط وعدم الخروج عن السياق.
- **الاستقرار والأمان**: التركيز على أمان الأسرار (Secrets)، إعداد بيئة محلية موثوقة، وبناء خدمات قابلة للاختبار.
- **التكامل المستمر (CI)**: الحفاظ على جودة الكود من خلال طلبات CI (GitHub Actions).

## 📋 القواعد الإجرائية (Operational Rules)
- **أمان البيانات**: لا تقم أبداً بتضمين مفاتيح API أو أسرار في ملفات الكود أو الـ Commits. استخدم ملفات `.env` أو `firebase-applet-config.json`.
- **التعديلات الوظيفية**: لا تقم بتغييرات هيكلية واسعة دون موافقة صرحية (مثل إعادة هيكلة الخدمات).
- **التوثيق**: حافظ على تحديث ملف `README.md` عند إضافة ميزات جديدة.
- **التواصل**: يجب أن تكون الردود موجزة، عملية، وبالتواصل باللغة العربية الفصحى.

## 🛠️ الخطوات التنفيذية القياسية
1. **فحص الأسرار**: التأكد من عدم وجود بيانات حساسة مكشوفة.
2. **فصل المنطق**: عزل عمليات الوصول للبيانات في `src/services/*` (عند الحاجة).
3. **الجودة**: تشغيل `npm run lint` و `npm test` قبل إنهاء أي مهمة.
4. **الأمن**: تحديث `firestore.rules` للتوافق مع أي تغييرات في هيكلة البيانات.

## 📝 تسميات الالتزام (Commit Naming)
يجب استخدام بادئات واضحة:
- `fix/security`: لإصلاحات أمنية.
- `chore(ci)`: لتحديثات التكامل المستمر.
- `refactor(service)`: لإعادة هيكلة الخدمات.
- `feat`: لإضافة ميزات جديدة.
